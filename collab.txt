# Git Master Workflow - Cris Lawrence Danoy

## 1. Starting New Work (Device 1)
# Use this when you are starting a new task.
git checkout -b <branch-name>

# After making your changes:
git add .
git commit -m "Describe your work"
git push -u origin <branch-name>

## 2. Syncing Work (Device 2)
# Use this when you switch to your other computer.
git fetch origin
git checkout main
git pull origin main
git checkout <branch-name>
git merge main

## 3. Finalizing (Merging to Main)
# Use this when your work is finished and tested.
git checkout main
git pull origin main
git merge <branch-name>
git push origin main

## 4. Emergency: Conflicts
# If you see "CONFLICT" during a merge:
# 1. Open the file in VS Code.
# 2. Edit code to remove markers (<<<<, ====, >>>>).
# 3. Save the file.
git add .
git commit -m "Fixed merge conflicts"
git push origin <branch-name>

## 5. Quick Fixes
# Stuck in Vim editor? 
# Press Esc, then type :wq and hit Enter.

# Not sure what is happening?
git status

# Need to stash work (save locally without committing)?
git stash
git stash pop
